## 2. フロントエンド要件定義（React Native / Expo）

**技術選定理由:** React Native / Expoの採用により、iOS/Androidでネイティブなパフォーマンスとスムーズなアニメーションを実現し、「成仏演出」や「お焚き上げ」などの凝ったUXを提供します。また、開発チームが4名体制でフロントとバックを分離するため、JavaScript/TypeScriptで開発できるReact Nativeは、開発効率の面でも優れています。

### 2.1. 主な機能要件

| **ID** | **機能名** | **詳細** | **備考** |
| --- | --- | --- | --- |
| FE-01 | **タイムライン表示** | 愚痴投稿を新しい順に表示。スクロール時の軽快な動作を担保。 | プルリフレッシュ対応。 |
| FE-02 | **投稿機能** | テキスト入力のみ（文字数制限あり）。「毒レベル」の自己申告機能（Lv.1〜5の選択）。 | 画像/動画の投稿は不可。匿名性を担保。 |
| FE-03 | **「わかる…」ボタン** | 投稿ごとに1ユーザー1回のみ押下可能。押下時に即座にカウントがリアルタイムで更新されること。 | 共感スタンプ機能も実装。 |
| FE-04 | **成仏演出** | 「わかる…」がある一定数（例：50件）に達した際、タイムライン上から**アニメーションと共に**投稿が消滅する演出を実行。 | 演出はカスタマイズ可能とする (煙、昇天など)。 |
| FE-05 | **フィルター機能** | 「毒レベル」によるフィルタリング（例: Lv.3以上のみ表示）、「未成仏」投稿のみ表示などを可能にする。 |  |
| FE-06 | **お焚き上げイベントUI** | イベント専用タブを設け、集まっている愚痴の数やテーマを表示。深夜0時の「お焚き上げ」時に特別な**全画面アニメーション**を表示。 | ネイティブアプリならではのリッチな演出を要求。 |
| FE-07 | **大怨霊討伐UI(将来)** | 大怨霊の画像と巨大なHPゲージを視覚的に表示。ユーザーの投稿/共感によるHP減少をリアルタイムで反映。 | 今回の作成は必要なし |
| FE-08 | **投票機能（将来）** | 投稿作成時に2〜3択の投票を追加できるUI。投票結果をリアルタイムでパーセンテージ表示。 | 今回の作成は必要無し |

2.2. 非機能要件（フロントエンド）

| **項目** | **要件** | **備考** |
| --- | --- | --- |
| **パフォーマンス** | スムーズなスクロール（60fps以上）と高速な画面遷移。特にタイムラインの読み込み速度を重視。 |  |
| **リアルタイム性** | 「わかる…」カウント、「成仏」、「大怨霊HP」の更新は、WebSocketまたはPush通知で即座に反映。 |  |
| **エラー処理** | 投稿失敗、API接続エラー時は、ユーザーに明確なフィードバックを返す。 |  |
